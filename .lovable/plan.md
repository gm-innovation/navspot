

# Fix Recovery Script: Remove Escaped Dollar Signs

## Problem

The recovery `.rsc` file generated by `navspot-recovery` uses `\$` (backslash-escaped dollar signs) for all RouterOS variables -- e.g. `\$tokenExists`, `\$hs`, `\$hp`. This escaping is only correct inside `source=""` parameters of `/system script add`. For standalone `.rsc` files imported via `/import`, bare `$` is required. RouterOS chokes on `\$` at line 11 column 12.

## Root Cause

In the JavaScript template literal inside `generateRecoveryScript()`, all RouterOS variable references use `\\$` which produces `\$` in the output. They should use plain `$` (which works fine in JS template literals since none are followed by `{`).

## Fix

**File:** `supabase/functions/navspot-recovery/index.ts` (lines 160-229)

Replace every `\\$` that precedes a RouterOS variable name with plain `$`:

| Line | Before | After |
|------|--------|-------|
| 171 | `\\$tokenExists` | `$tokenExists` |
| 195 | `\\$installFile` | `$installFile` |
| 210 | `\\$hs` (x3), `\\$pN` | `$hs`, `$pN` |
| 211 | `\\$hp` | `$hp` |
| 212 | `\\$hp` | `$hp` |
| 213 | `\\$hp` | `$hp` |
| 214 | `\\$hp` (x2) | `$hp` |

**Important:** Do NOT touch `${syncToken}`, `${VERSION}`, `${DEPLOYED_AT}`, `${scriptsUrl}` -- those are JavaScript interpolations and must stay as-is.

## Verification

After deploy, download the recovery script and confirm:
- Line 11 reads `:if ([:len $tokenExists] = 0) do={` (bare `$`)
- All other variable references use bare `$`
- Import on RouterOS completes without syntax errors
